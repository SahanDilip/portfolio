Laptop Price Predictor — ML-powered Price Estimator

A data-driven web application that estimates laptop prices using machine learning. The project includes dataset exploration, feature engineering, model training (scikit-learn), a pickled production model, and a Flask-based web UI + REST API for predictions. The app converts predictions to LKR and exposes an endpoint for programmatic use.

Key Features

  • Dataset & EDA: Cleaned and analyzed a public laptop price dataset; performed visualizations and statistical checks in Jupyter notebooks.
  • Feature Engineering: Numeric normalization and one-hot encoding of categorical fields (brand, CPU, GPU, OS, type), binary features for touchscreen/IPS, and weight/ram handling.
  • Model Training & Evaluation: Built and compared regression models using scikit-learn, selected and tuned the best model, and evaluated using RMSE and R² metrics.
  • Model Persistence: Exported the trained model to predictor.pickle for production usage and fast inference.
  • Flask Web App & REST API: Lightweight Flask app exposing a /api/predict POST endpoint and a simple web UI (index.html) for interactive predictions.
  • Currency Conversion: Prediction output converted from EUR (model units) to LKR before returning to the user.
  • Notebooks & Reproducibility: Jupyter notebook included for exploratory data analysis and model training steps, enabling reproducibility.
  • Easy Integration: API accepts form-data and can be called from other services or from a frontend form for immediate predictions.

Tech Stack: Python, Flask, scikit-learn, pandas, NumPy, Jupyter, HTML/CSS, JavaScript

NanoProcessor — 4-bit Educational CPU

A compact, hand-designed 4-bit processor implemented in VHDL and validated with testbenches and an FPGA (BASYS3). Designed to demonstrate datapath/control design, modular component integration, and hardware verification.

Key Features

  • Small Instruction Set: Implements 4 instructions — MOVI R, d, ADD Ra, Rb, NEG R, and JZR R, d — ideal for teaching control & datapath design.
  • Modular Component Design: Separate, reusable modules for the 4-bit add/sub unit, 3-bit adder, program counter, and multiple multiplexers, enabling clear separation of concerns and easy testing.
  • Register Bank: 4-bit register bank with read/write control for general-purpose registers used by the instruction set.
  • Program ROM & Instruction Decoder: ROM-based instruction storage with a dedicated decoder to produce control signals for the datapath.
  • Clocking & Slow Clock: Built-in slow clock generation for step-by-step hardware debugging and synchronization with testbenches and FPGA board buttons/LEDs.
  • Testbenches & Simulation: Extensive VHDL testbenches (e.g., adder and CPU-level tests) run with XSIM/ModelSim, providing waveform outputs for functional verification.
  • FPGA Implementation: Synthesized and implemented with Xilinx Vivado; bitstream generation and board testing performed on the BASYS3 development board.
  • Reports & Artifacts: Timing, utilization, and route reports generated during implementation (timing summary, utilization reports, .bit file, and TCL scripts present in the project outputs).

Tech Stack: VHDL, Xilinx Vivado (synthesis & implementation), XSIM/ModelSim (simulation), BASYS3 FPGA, TCL scripts & testbenches

Designed Components

  • 4-bit Add/Subtract Unit — supports add and subtract operations, used for ALU behavior including NEG (two's complement).
  • 3-bit Adder — used for program counter increments and address arithmetic.
  • 3-bit Program Counter — holds instruction addresses and supports conditional jumps.
  • 2-way 3-bit Multiplexer — select between address sources or small control paths.
  • 2-way 4-bit Multiplexer — used in datapath to select immediate vs register data.
  • 8-way 4-bit Multiplexer — selects register outputs for ALU inputs or write-back.
  • Register Bank — small register file for the CPU registers, supporting read/write ports.
  • Program ROM — stores the instruction stream (assembled machine code for the 4-instruction ISA).
  • Instruction Decoder — decodes opcodes and generates control signals for datapath elements.
  • Slow Clock — optional slowed clock for manual stepping during debugging on-board.

MediBox — Automated Medicine Dispenser

Key Features

  • Automated Medicine Dispensing: Servo-controlled dispenser that rotates to drop selected medicine doses based on light-guided selection and scheduling.
  • Scheduled Alarms & Reminders: Multi-alarm scheduler with NTP-based time sync, audible buzzer and LED notifications for medicine times; configurable alarms via onboard menu.
  • Environment Monitoring & Alerts: Real-time temperature and humidity sensing (DHT22) with visual warnings and display messages when thresholds are exceeded.
  • Light-based Selection & Control: Dual LDR sensors determine light-intensity and guide servo angle calculation for selective dispensing (supports multiple medicine slots).
  • Local UI & Controls: OLED display (SSD1306) + pushbutton menu for configuring timezone, alarms, and interacting with the device directly.
  • Remote Telemetry & Control (MQTT): Wi‑Fi connectivity and MQTT support (PubSubClient) to publish sensor data (temperature, humidity, light) and receive control topics for parameters like servo offsets.
  • Hardware Safety & Feedback: Buzzer, alarm LED, and fail-safe button (cancel) to stop alarms; temperature/humidity warning LEDs for immediate onsite alerts.
  • Simulation & Prototyping: Full Wokwi simulation files and circuit diagram included for easy prototyping and demonstration.

Tech Stack: ESP32 (Arduino/C++), Adafruit GFX & SSD1306, DHTesp, ESP32Servo, PubSubClient (MQTT), Wokwi simulation
